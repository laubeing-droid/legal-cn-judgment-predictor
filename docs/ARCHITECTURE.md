# 架构设计

## 设计哲学

**纯 Prompt 驱动，零模型训练。** 所有法律推理能力来自 LLM + 结构化知识注入，不依赖微调模型。

## 核心流程

```
┌─────────────────────────────────────────────────────────┐
│                      输入：案件事实描述                       │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ① extract/fact-elements.md                              │
│     主体识别 → 行为事实 → 结果事实 → 争议焦点 → 案由判定       │
│     输出：结构化事实要素                                    │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ② prompts/plaintiff.md  +  prompts/defendant.md         │
│     原告/公诉方构建论证      被告/辩护方构建抗辩              │
│     每个主张须：事实 + 法条 + 证据支撑点                      │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ③ debate/rounds.md                                      │
│     第1轮：事实主张与抗辩                                   │
│     第2轮：法律适用辩论                                     │
│     第3轮：综合辩论与最后陈述                                │
│     法官每轮归纳争议焦点                                    │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ④ prompts/judge.md                                      │
│     事实认定 → 法律适用 → 裁判推理 → 结果预测                  │
│     引用 reasoning-template-zh.md 中国法推理链             │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ⑤ predict/ (civil.md | criminal.md | admin.md)          │
│     胜败方向 + 金额/刑期区间 + 置信度 + 关键因子              │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ⑥ evaluate/metrics.md                                   │
│     法条准确性 / 逻辑完整性 / 阻断过滤 / 结果一致性 / 推理深度   │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  ⑦ report/template.md                                    │
│     结构化预测报告：法条 + 结果 + 置信度 + 推理链 + 自评        │
└─────────────────────────────────────────────────────────┘
```

## 模块职责

| 模块 | 文件 | 职责 | 输入 | 输出 |
|:-----|:-----|:-----|:-----|:-----|
| **入口** | SKILL.md | 编排全流程 | 用户案件描述 | 预测报告 |
| **抽取** | extract/ | 结构化事实 | 自然语言 | 结构化要素 |
| **角色** | prompts/ | 三角色论证 | 结构化事实 | 各方论证 |
| **辩论** | debate/ | 编排辩论 | 各方论证 | 争议焦点归纳 |
| **预测** | predict/ | 判决预测 | 辩论结果 | 法条+结果+置信度 |
| **评估** | evaluate/ | 自评 | 预测结果 | 自评报告 |
| **报告** | report/ | 格式化输出 | 全流程结果 | 结构化报告 |
| **联动** | mcp/ | MCP 调用策略 | — | — |

## 案由覆盖

| 案由类别 | 预测模板 | 推理方法 |
|:---------|:---------|:---------|
| 民事 | predict/civil.md | 请求权基础分析法 |
| 刑事 | predict/criminal.md | 犯罪构成要件检视 |
| 行政 | predict/admin.md | 行政行为合法性审查 |
