# 多维度自评

## 目标
每轮预测结束后，对输出结果进行多维度自评。发现缺陷自动标记，触发修正或标注不确定性。

## 自评维度

### 一、法条准确性
- [ ] 引用的法条编号是否正确？（核对 law-semantic-tree.json）
- [ ] 法条是否为现行有效版本？（核对 law-versions.json）
- [ ] 法条适用于当前案由？
- [ ] 是否存在法条引用遗漏？

### 二、逻辑完整性
- [ ] 大前提→小前提→结论 推理链是否完整？
- [ ] 构成要件是否逐项检视？
- [ ] 抗辩事由是否逐一回应？
- [ ] 是否存在逻辑跳跃？

### 三、阻断概念过滤
- [ ] 输出中是否包含 blocking-list.md 中的阻断概念？
- [ ] 是否误用了美国法制度名称？
- [ ] 是否引用了境外法域法律？

### 四、结果一致性
- [ ] 事实认定与法律适用是否一致？
- [ ] 定量结论是否与推理过程一致？
- [ ] 关键因子是否与结果方向匹配？

### 五、推理深度
- [ ] 是否仅做了表面分析？
- [ ] 是否考虑了抗辩方最强论点？
- [ ] 是否标注了证据缺口？

## 自评格式
```
[自评报告]

✅ 法条准确性：全部正确 / ⚠️ [具体问题]
✅ 逻辑完整性：推理链完整 / ⚠️ [具体缺失]
✅ 阻断过滤：未检出阻断概念 / ❌ [检出概念] → 已修正
✅ 结果一致性：一致 / ⚠️ [不一致点]
✅ 推理深度：充分 / ⚠️ [不足点]

总体自评：🟢 可信度较高 / 🟡 需补充信息 / 🔴 存在严重缺陷
```
