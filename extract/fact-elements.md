# 事实要素结构化抽取

## 目标
将自然语言案件描述抽取为结构化事实要素，供后续角色论证和法律适用使用。

## 抽取维度

### 一、主体识别
| 要素 | 说明 |
|:-----|:-----|
| 原告/公诉方 | 自然人/法人/检察院 + 身份标识 |
| 被告/辩护方 | 自然人/法人 + 身份标识 |
| 第三人（如有） | 有/无独立请求权 |
| 代理人/辩护人 | 律师事务所+律师（如有） |

### 二、行为事实
| 维度 | 民事案件 | 刑事案件 |
|:-----|:---------|:---------|
| 主观方面 | 善意/恶意/过失/无过错 | 故意/过失/意外 |
| 客观行为 | 作为/不作为的具体描述 | 行为类型+手段+工具 |
| 时间地点 | 行为发生精确时间+地点 | 同左 |
| 因果关系 | 行为→结果 链条 | 同左 |

### 三、结果事实
| 类型 | 描述 |
|:-----|:-----|
| 财产损失 | 金额+计算依据 |
| 人身损害 | 伤害类型+等级+医疗记录 |
| 权利侵害 | 具体权利类型（物权/债权/人格权/知识产权） |
| 社会危害（刑事） | 情节严重性+社会影响 |

### 四、争议焦点提取
从事实对比中自动识别：
1. 事实争议点（双方对事实描述不一致之处）
2. 法律争议点（对法律适用的不同理解）
3. 证据争议点（关键证据是否存在/可采信）

### 五、案由初步判定
基于事实要素匹配案由：
- 合同纠纷 / 侵权纠纷 / 物权纠纷 / 婚姻家庭 / 继承 / 劳动争议 / 知识产权 / 公司纠纷 / 行政争议 / 刑事罪名

## 输出格式
```

### 行为事实

2. 法律争议：[]

```
